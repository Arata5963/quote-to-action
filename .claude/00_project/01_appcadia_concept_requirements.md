# mitadake? - YouTube動画から学びを行動に変える

## 1. プロダクトコンセプト

### 1.1 サービス概要

**mitadake?（みただけ？）**は、YouTube動画から得た学びを具体的な「アクションプラン」に変換して実践を支援するプラットフォームです。

「見ただけ」で終わらせない。YouTube動画の学びを行動に変え、「実践して成長」へと体験を変革します。

### 1.2 ビジョン

> 「見ただけで終わらせない」
>
> YouTube動画から得た学びを行動に変え、小さな一歩の積み重ねで大きな変化を生み出す

### 1.3 解決する課題

#### 現状の問題
- YouTube動画で得た「これは大切だ」という気づきが日常に埋もれてしまう
- 動画視聴と実践のタイミングに時間的ギャップがある
- 「良い動画を見た」という漠然とした記憶しか残らない
- 学びを行動に移すための具体的な仕組みがない

#### ソリューション
- YouTube URLを即座に記録できるシンプルなインターフェース
- 動画視聴直後に具体的なアクションプランを設定
- リマインダー機能による実践促進
- コミュニティ機能による継続支援

## 2. ターゲットユーザー

### 2.1 プライマリターゲット

**忙しい現代人で成長意欲のある層（20代〜40代）**

- 自己啓発や学習に関心があるビジネスパーソン
- 日々の気づきを大切にしたい人
- 複雑な操作より「簡単で今すぐ終わる」ことを重視
- スモールステップで着実に成長したい人

### 2.2 ユーザーペルソナ

**ペルソナ1: 成長志向のビジネスパーソン（30代）**
- YouTube動画で自己啓発やスキルアップの動画をよく見るが、実践まで至らない
- 「良い動画を見た」で終わってしまうことにもどかしさを感じている
- 継続的な成長を望んでいるが、大きな変化は難しいと感じている

**ペルソナ2: 学びを活かしたい学生・若手社会人（20代）**
- YouTubeで様々な情報に触れる機会が多い
- インプットは多いがアウトプットの機会が少ない
- 小さな一歩から始められる成長ツールを求めている

## 3. サービスの利用フロー

### 3.1 基本的な使い方

```
STEP 1: YouTube動画を保存
↓ YouTube URLを入力
↓ 動画タイトル・チャンネル名を自動取得
↓ YouTube公式カテゴリで分類

STEP 2: アクションプラン設定
↓ 具体的で実行可能な小さな行動を決定
↓ 例：「階段を使う」「15分読書する」
↓ リマインダー日時を設定（任意）

STEP 3: 実践と記録（Achievement）
↓ 達成ボタンで記録
↓ 達成感の可視化
```

### 3.2 継続を促す仕組み

- **リマインダー機能**: 設定した日時にメール通知
- **コミュニティ機能**: いいね、コメントによる相互励まし
- **達成状況の可視化**: 達成済み/未達成の投稿を管理

## 6. データモデル

### 6.1 主要エンティティ

```ruby
User
├── id: bigint (PK)
├── email: string (unique, not null)
├── name: string
├── avatar: string
├── provider: string (OAuth)
├── uid: string (OAuth)
└── has_many: posts, reminders, achievements, comments, likes

Post
├── id: bigint (PK)
├── user_id: bigint (FK)
├── youtube_url: string (not null)
├── youtube_title: string (自動取得)
├── youtube_channel_name: string (自動取得)
├── action_plan: text (1-100文字)
├── category: integer (enum, default: 6)
├── achieved_at: datetime
└── has_one: reminder
└── has_many: achievements, comments, likes

Reminder
├── id: bigint (PK)
├── user_id: bigint (FK)
├── post_id: bigint (FK)
├── remind_at: datetime (not null)
└── unique: [user_id, post_id]

Achievement
├── id: bigint (PK)
├── user_id: bigint (FK)
├── post_id: bigint (FK)
├── achieved_at: date (not null, default: CURRENT_DATE)
└── unique: [user_id, post_id]

Comment
├── id: bigint (PK)
├── user_id: bigint (FK)
├── post_id: bigint (FK)
└── content: string (not null)

Like
├── id: bigint (PK)
├── user_id: bigint (FK)
├── post_id: bigint (FK)
└── unique: [user_id, post_id]
```

## 7. UI/UXデザイン

### 7.1 デザイン原則

1. **シンプル**: 3タップ以内で主要操作完了
2. **直感的**: アイコンと色による視覚的フィードバック
3. **モチベーション重視**: 達成感を演出する要素
4. **モバイルファースト**: スマートフォンでの使いやすさ優先

- プライマリ: 黒 (#000000) - 力強さと決意
- アクセント: イエロー (#FACC15) - エネルギーと閃き
- 成功: グリーン (#10B981) - 達成と成長
- 背景: グレー (#F3F4F6) - 落ち着きと集中

### 7.3 主要画面

1. **投稿一覧**: カード形式、YouTubeサムネイル表示
2. **投稿フォーム**: YouTube URL入力、アクションプラン設定、リマインダー設定
3. **投稿詳細**: YouTube埋め込み、達成ボタン、コメント
4. **マイページ**: プロフィール編集、投稿一覧
